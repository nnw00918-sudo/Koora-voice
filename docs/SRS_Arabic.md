## 1. المقدمة

### 1.1 الغرض من الوثيقة
تهدف هذه الوثيقة إلى تحديد المتطلبات الوظيفية وغير الوظيفية لتطبيق "صوت الكورة" (Koora Voice)، وهو تطبيق تواصل اجتماعي متخصص في مجال كرة القدم يوفر غرف صوتية ومرئية للمستخدمين.

### 1.2 نطاق المشروع
**اسم التطبيق:** صوت الكورة (Koora Voice)  
**نوع التطبيق:** تطبيق ويب تقدمي (PWA) + تطبيق iOS  
**الفئة المستهدفة:** محبي كرة القدم في الوطن العربي  
**اللغات المدعومة:** العربية والإنجليزية

### 1.3 التعريفات والاختصارات

| الاختصار | التعريف |
|----------|---------|
| PWA | تطبيق ويب تقدمي (Progressive Web App) |
| API | واجهة برمجة التطبيقات |
| JWT | رمز JSON للمصادقة |
| WebSocket | بروتوكول اتصال ثنائي الاتجاه |
| RTL | من اليمين لليسار (Right to Left) |
| CRUD | إنشاء، قراءة، تحديث، حذف |

### 1.4 المراجع
- وثائق React.js الرسمية
- وثائق FastAPI
- وثائق MongoDB
- وثائق Agora SDK

## 2. الوصف العام

### 2.1 منظور المنتج
تطبيق "صوت الكورة" هو منصة تواصل اجتماعي متخصصة تجمع محبي كرة القدم في غرف صوتية ومرئية تفاعلية، مع إمكانية المحادثة النصية وتبادل الهدايا الافتراضية.

### 2.2 وظائف المنتج الرئيسية
```
┌────────────────────────────────────────────────────────┐
│                    صوت الكورة                          │
├────────────────────────────────────────────────────────┤
│  ┌──────────┐  ┌──────────┐  ┌──────────┐            │
│  │  غرف    │  │  دردشة  │  │  بث     │            │
│  │ صوتية   │  │  نصية   │  │  مباشر  │            │
│  └──────────┘  └──────────┘  └──────────┘            │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐            │
│  │  هدايا  │  │ إعلانات │  │  ملفات  │            │
│  │ افتراضية│  │  عامة   │  │  شخصية  │            │
│  └──────────┘  └──────────┘  └──────────┘            │
└────────────────────────────────────────────────────────┘
```

### 2.3 خصائص المستخدمين

| نوع المستخدم | الوصف | الصلاحيات |
|--------------|-------|-----------|
| زائر | مستخدم غير مسجل | عرض الغرف العامة فقط |
| مستخدم عادي | مستخدم مسجل | الانضمام للغرف، الدردشة، إرسال الهدايا |
| مشرف (Mod) | مشرف على غرفة | كتم المستخدمين، حذف الرسائل |
| قائد (Leader) | قائد الغرفة | صلاحيات المشرف + إدارة المقاعد |
| أدمن (Admin) | مدير الغرفة | صلاحيات القائد + تعيين الأدوار |
| مالك الغرفة | منشئ الغرفة | كامل الصلاحيات على الغرفة |
| مالك التطبيق | مدير النظام | كامل الصلاحيات على التطبيق |

### 2.4 بيئة التشغيل
- **المتصفحات:** Chrome, Safari, Firefox, Edge
- **الأجهزة:** هواتف ذكية، أجهزة لوحية، حواسيب
- **أنظمة التشغيل:** iOS, Android, Windows, macOS

### 2.5 القيود التصميمية
- دعم اللغة العربية (RTL)
- تصميم متجاوب (Responsive Design)
- توافق مع معايير PWA
- أداء عالي مع تحميل سريع

- **النوع:** Agora RTC SDK
